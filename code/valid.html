Banking System Validation Patterns

1. Name Validation

Rules:

  • Length: 3 – 50
  • Alphabets
  • Spaces allowed
  • Maximum 6 dots
  • No continuous dots
/^(?=.{3,50}$)(?=(?:[^.]*\.){0,6}[^.]*$)[A-Za-z]+(?:[ .][A-Za-z]+)*\.?$/

2. Email Validation

Supports standard email symbols and prevents continuous dots.

^(?=.{5,254}$)[A-Za-z0-9!#$%&'*+/=?^_`{|}~-]+(?:\.[A-Za-z0-9!#$%&'*+/=?^_`{|}~-]+)*@
[A-Za-z0-9-]+(?:\.[A-Za-z]{2,})+$

3. Date Validation (DD-MM-YYYY)

Supports:

  • Leap years
  • Correct month days
  • Year range 1900–2099
^(?:(?:31-(?:0?[13579]|1[02])-(?:19|20)\d\d)|
(?:(?:29|30)-(?:0?[13-9]|1[0-2])-(?:19|20)\d\d)|
(?:29-0?2-(?:(?:19|20)(?:0[48]|[13579][26]|[2468][048])|2000))|
(?:(?:0?[1-9]|1[0-9]|2[0-8])-(?:0?[1-9]|1[0-2])-(?:19|20)\d\d))$

4. Password

Rules:

  • 8 – 16 characters
  • Uppercase required
  • Lowercase required
  • Number required
  • Special character required
^(?=.*[A-Z])(?=.*[a-z])(?=.*\d)(?=.*[@#$%^&+=!]).{8,16}$

5. Amount

Supports decimal up to 2 places.

^-?(?:0|[1-9]\d{0,11})(?:\.\d{1,2})?$

6. Account Number

Typical bank account number length.

^[0-9]{9,18}$

7. IFSC Code

^[A-Z]{4}0[A-Z0-9]{6}$
Example: 
SBIN0001234

8. Aadhaar Number

^[2-9][0-9]{11}$

9. PAN Number

^[A-Z]{5}[0-9]{4}[A-Z]$
Example: 
ABCDE1234F

10. Mobile Number (India)

^[6-9][0-9]{9}$

11. Tenure Validation

Months (1 – 360)

^(?:[1-9]|[1-9][0-9]|[12][0-9]{2}|3[0-5][0-9]|360)$

Years (1 – 30)

^(?:[1-9]|[12][0-9]|30)$

12. Card Number (Generic)

Supports most debit and credit cards.

^\d{13,19}$

13. Visa Card

Starts with 4

^4(?:\d{12}|\d{15}|\d{18})$
Example: 
4111111111111111
4012888888881881

14. MasterCard

Range: 51–55 and 2221–2720

^(5[1-5]\d{14}|2(22[1-9]\d{11}|2[3-9]\d{12}|[3-6]\d{13}|7[01]\d{12}|720\d{12}))$

15. American Express

Starts with 34 or 37

^3[47]\d{13}$

16. RuPay Card

Starts with:

  • 60
  • 65
  • 81
  • 82
  • 508
  • 353
  • 356
^(?:60\d{14}|65\d{14}|81\d{14}|82\d{14}|508\d{13}|353\d{13}|356\d{13})$

17. Combined Card Validation

Supports:

  • Visa
  • MasterCard
  • American Express
  • Discover
  • RuPay
^(?:4(?:\d{12}|\d{15}|\d{18})|
5[1-5]\d{14}|
2(?:22[1-9]\d{11}|2[3-9]\d{12}|[3-6]\d{13}|7[01]\d{12}|720\d{12})|
3[47]\d{13}|
6(?:011|5\d{2})\d{12}|
(?:81\d{14}|82\d{14}|508\d{13}|353\d{13}|356\d{13}))$

Supported Card Networks

Card Prefix
Visa 4
MasterCard 51–55, 2221–2720
American Express 34, 37
Discover 6011, 65
RuPay 60, 65, 81, 82, 508, 353, 356

Note

In real banking systems, card numbers are also validated using Luhn Algorithm and BIN range checks. Regex only validates the structure.
