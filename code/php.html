PHP Basics

PHP (Hypertext Preprocessor) is a server-side scripting language used to build dynamic web applications. PHP code runs on the server and outputs HTML to the browser.

1. What is PHP?

PHP is a widely-used, open-source scripting language especially suited for web development. It can interact with databases, handle forms, manage sessions, and generate dynamic web pages.

  • Server-side scripting language
  • Embedded inside HTML
  • Executes on the server, not in the browser
  • Supports databases like MySQL, PostgreSQL, SQLite
  • Easy to learn and widely used

7. PHP Forms

Forms in PHP are used to collect user input from web pages. Data can be sent using GET or POST methods.

7.1 GET Method

Data is sent via URL. Visible in browser. Suitable for non-sensitive data.

<!DOCTYPE html>
<html>
<body>

<form method="get" action="process.php">
  Name: <input type="text" name="username"><br>
  <input type="submit" value="Submit">
</form>

</body>
</html>

7.2 POST Method

Data is sent in HTTP request body. More secure. Suitable for passwords and sensitive data.

<form method="post" action="process.php">
  Password: <input type="password" name="password"><br>
  <input type="submit" value="Submit">
</form>

7.3 Form Validation

Always validate user input before using it.

7.4 $_REQUEST Superglobal

$_REQUEST contains data from GET, POST, and COOKIE. Can access any form data, but it's better to use $_GET or $_POST specifically.

7.5 Summary

  • Use GET for non-sensitive data, URL parameters.
  • Use POST for sensitive data and larger data.
  • Always validate form inputs using PHP.
  • Use $_GET, $_POST over $_REQUEST for clarity and security.

14. PHP & HTML Integration

PHP can be integrated with HTML using include, require, and templates for modular code.

14.1 include

Includes another PHP file. Gives a warning if file not found but continues execution.

<?php
include 'header.php';
?>

<h1>Welcome</h1>

<?php
include 'footer.php';
?>

14.2 require

Includes another PHP file. Throws a fatal error if file not found.

<?php
require 'config.php';
?>

14.3 include_once / require_once

Ensures the file is included only once to prevent redeclaration errors.

<?php
include_once 'functions.php';
require_once 'config.php';
?>

14.4 Templates

Separates HTML layout from PHP logic for clean code.

<!-- header.php -->
<header>
  <h1>My Website</h1>
</header>

<!-- index.php -->
<?php include 'header.php'; ?>
<main>
  <p>Welcome to my site!</p>
</main>
<?php include 'footer.php'; ?>

<!-- footer.php -->
<footer>
  <p>Footer content</p>
</footer>

14.5 Summary

  • include and require allow modular code.
  • _once prevents multiple inclusions.
  • Templates help separate logic from presentation.
