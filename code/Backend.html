HTTP (HyperText Transfer Protocol)

HTTP is a communication protocol used by clients and servers to exchange data over the internet.

Whenever a browser, mobile application, desktop application, or API communicates with a server, it usually uses HTTP or HTTPS. 


Client
   ↓ HTTP Request
Server
   ↓ HTTP Response
Client

Why HTTP Exists

Computers need a standard way to communicate.

HTTP defines:

  • How requests are sent.
  • How responses are returned.
  • How data is formatted.
  • How errors are reported.

Key Points to Remember

  • HTTP is the communication protocol used by web applications.
  • Clients send requests and servers send responses.
  • Methods define actions such as GET, POST, PUT and DELETE.
  • Headers contain metadata about requests and responses.
  • JSON is the most common API data format.
  • Status codes indicate request results.
  • HTTP is stateless.
  • HTTPS uses TLS encryption for secure communication.

HTTP Security and HTTPS Security

HTTP and HTTPS are communication protocols used between clients and servers.

HTTPS secures communication using encryption, while HTTP sends data as plain text.

How Attackers Can Access Data Even With HTTPS

HTTPS protects network traffic, but attackers may use other attack methods.

Attack 1 - SQL Injection

If user input is directly placed into SQL queries, attackers may manipulate the query. 


Input:

admin' OR '1'='1

Protection:

  • Parameterized Queries.
  • Prepared Statements.
  • Input Validation.

Attack 2 - Cross Site Scripting (XSS)

Attackers inject JavaScript into web pages. 


<script>
alert('XSS');
</script>

Protection:

  • Output Encoding.
  • Input Validation.
  • Content Security Policy.

Attack 3 - Session Hijacking

An attacker steals a user's session token and impersonates that user. 


User Login
    ↓
Session Cookie
    ↓
Attacker Obtains Cookie
    ↓
Uses Cookie

Protection:

  • HTTPS.
  • Secure Cookies.
  • HttpOnly Cookies.
  • Session Expiration.

Attack 4 - Brute Force Attack

Attackers try thousands of passwords until one works.

Protection:

  • Rate Limiting.
  • Account Locking.
  • MFA.
  • Strong Password Policies.

Attack 5 - Stolen Database

Attackers may compromise the server and steal the database.

HTTPS cannot protect against this because the attack occurs after data reaches the server.

Protection:

  • Password Hashing.
  • Encryption at Rest.
  • Access Controls.

Attack 6 - Malware on User Device

If the user's computer is infected, malware may steal information before HTTPS encryption occurs. 


Keyboard
    ↓
Malware Reads Input
    ↓
HTTPS Encryption Happens Later

HTTPS cannot prevent malware installed on the user's device.
