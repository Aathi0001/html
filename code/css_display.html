CSS Display Property

→ What is display?

The display property determines how an element is displayed (rendered) in the document flow.

It is one of the most important properties in CSS for layout.

→ Syntax:
display: value;
→ Common Values of display:
  • block → The element takes full width and starts on a new line. (e.g., <div>, <p>)
  • inline → The element only takes as much width as it needs. It does not start on a new line. (e.g., <span>)
  • inline-block → Behaves like inline but allows setting width and height.
  • none → Hides the element completely (no space is reserved).
→ Flex and Grid Display:
  • flex → Enables Flexbox layout for flexible item arrangement.
  • inline-flex → Like flex, but behaves like an inline element.
  • grid → Enables Grid layout for 2D layout structure (rows & columns).
  • inline-grid → Like grid, but behaves like an inline element.
→ Table Display:
  • table → Behaves like an HTML <table> element.
  • table-row → Behaves like a <tr> element.
  • table-cell → Behaves like a <td> element.
→ Other Values:
  • contents → Makes the element disappear, but keeps its children visible (without creating a box).
  • list-item → Behaves like a <li> element with bullet/number.
  • run-in → Rarely used; tries to run the element into the next block if possible.
→ Example:
div {
  display: inline-block;
}


Flexbox Layout

  • Flexbox is used to design layouts in a one-dimensional format (row or column).
  • It requires a flex container, and only its direct children become flex items.

Flex Container Properties

1. display: flex

  • This enables flex context and allows other flex properties to be used.
  • Example: 
    .container {
  display: flex;
}

2. justify-content

  • Aligns flex items along the main axis.
  • Syntax: justify-content: value;
  • Values:
    • flex-start – Items align at the start (default).
    • center – Items align at the center.
    • flex-end – Items align at the end.
    • space-between – Equal spacing between items, first and last stick to edges.
    • space-around – Equal spacing around items.
    • space-evenly – Equal spacing between and around items.

3. align-items

  • Aligns items along the cross axis.
  • Syntax: align-items: value;
  • Values:
    • flex-start – Items align at the start (default).
    • center – Items align at the center.
    • flex-end – Items align at the end.

4. flex-direction

  • Specifies the direction of the main axis.
  • Syntax: flex-direction: value;
  • Values:
    • row – Default. Left to right.
    • row-reverse – Right to left.
    • column – Top to bottom.
    • column-reverse – Bottom to top.

5. flex-wrap

  • Determines whether flex items wrap to the next line.
  • Syntax: flex-wrap: value;
  • Values:
    • nowrap – Default. No wrapping.
    • wrap – Items wrap to next line.
    • wrap-reverse – Items wrap in reverse order.

6. align-content

  • Aligns rows of wrapped items in the cross axis.
  • Syntax: align-content: value;
  • Values:
    • flex-start
    • center
    • flex-end
    • space-between
    • space-around
    • space-evenly

7. flex-flow

  • Shorthand for flex-direction and flex-wrap.
  • Syntax: flex-flow: flex-direction flex-wrap;
  • Example: 
    .container {
  flex-flow: row wrap;
}

8. column-gap / row-gap / gap

  • column-gap: Space between columns.
  • row-gap: Space between rows.
  • gap: Shorthand for both row and column gaps.
  • Example: 
    .container {
  gap: 20px 30px; /* row-gap column-gap */
}

flex-grow

Definition: Controls how much an item expands relative to others when extra space is available.

Grow:1
Grow:2
Grow:1

Output: The middle item (Grow:2) takes double the space of Grow:1 items. Ratio = 1:2:1.

flex-shrink

Definition: Controls how much an item shrinks relative to others when space is too small.

Shrink:0
Shrink:1

Output: The first item (Shrink:0) will NOT shrink below 200px, while the second item will shrink to fit.

flex-basis

Definition: Sets the initial size of the item before grow/shrink is applied.

Basis:100px
Basis:200px

Output: The first item starts at 100px, the second at 200px. After that, grow/shrink rules may apply.

Summary

  • flex-grow: Distributes extra space (expand).
  • flex-shrink: Distributes space reduction (shrink).
  • flex-basis: Starting size of the flex item.
  • flex: shorthand → flex: grow shrink basis;

Flex Item Properties

1. order

  • Changes the order of items (ascending).
  • Default order for all items is 0.
  • Example: 
    .item1 { order: 4; }
.item2 { order: 1; }
.item3 { order: 2; }
.item4 { order: 3; }

2. align-self

  • Overrides align-items for individual flex items.
  • Used to align a single item on the cross axis.
  • Values: flex-start, center, flex-end
  • Example: 
    .item2 {
  align-self: center;
}

3. flex-basis

  • Specifies the initial size (main axis) of a flex item.
  • Acts like width in row direction and height in column direction.
  • Example: 
    .item2 {
  flex-basis: 100px;
}

Extra Notes for Better Understanding

  • Main Axis is defined by flex-direction.
  • Cross Axis is perpendicular to the main axis.
  • Use flex-wrap: wrap if you want items to move to the next line on smaller screens.
  • Use gap for consistent spacing instead of margin.
  • align-items (container-wide) vs align-self (per item).


CSS Positions

  • The position property is used to control the placement of elements on a webpage.
  • It determines how an element is positioned in relation to its normal flow and other elements.
  • There are 5 types of positioning:
    • Static
    • Relative
    • Absolute
    • Fixed
    • Sticky
  • To control positioned elements, we use helper properties:
    • top
    • bottom
    • left
    • right

1. Static Positioning

  • This is the default position of all elements.
  • The element follows the normal document flow.
  • Helper properties (top, bottom, left, right) do not affect static elements.

2. Relative Positioning

  • Moves the element relative to its original position.
  • The space originally occupied by the element remains.
  • Other elements do not move into its place.
  • Example:
    • position: relative;
top: 20px;   /* Moves the element 20px down */
left: 10px;  /* Moves the element 10px right */

3. Absolute Positioning

  • Moves the element relative to the nearest positioned ancestor.
  • If no such ancestor exists, it is positioned relative to the viewport.
  • The element is removed from the normal document flow.
  • Other elements take its place.
  • Use z-index to control overlapping of absolute elements.
  • Example:
    • .parent {
  position: relative;
}
.child {
  position: absolute;
  top: 50px;
  left: 30px;
}

4. Fixed Positioning

  • Positioned relative to the viewport.
  • Does not move even when scrolling.
  • Helper properties define its position.
  • Example:
    • position: fixed;
top: 0;
left: 0;
width: 100%;
background: black;
color: white;

5. Sticky Positioning

  • Acts like relative until it reaches a scroll threshold, then becomes fixed.
  • Helper properties act like sticky conditions.
  • Example:
    • position: sticky;
top: 50px; /* Sticks 50px from the top when scrolling */

Important Notes

  • If an element has relative, absolute, fixed, or sticky positioning:
    • It is removed from normal document flow.
    • Floats on the z-axis (overlapping).
  • Avoid using both top and bottom or left and right together — may cause issues.


Grid

Grid lays out the webpage in a 2-dimensional format (rows and columns).

To use Grid:

  • You need a grid container using display: grid
  • All direct children of the grid container become grid-items

Grid Container Properties

1. display: grid

  • Makes the element a grid container
  • Allows use of all grid-related properties

2. grid-template-columns

  • Defines number and width of columns
  • Example: 
    grid-template-columns: 200px 200px 200px 200px;
    4 columns, each 200px wide
  • Other formats: 
    
grid-template-columns: 200px 100px 150px 80px;

     What it means:
    You are creating 4 columns with fixed widths:
    
Column 1 → 200px wide
Column 2 → 100px wide
Column 3 → 150px wide
Column 4 → 80px wide

     Use Case:
    Useful when you want to control the **exact size** of each column manually.
    

    ---------------------------------------------------------------------------------
    
grid-template-columns: 1fr 2fr 1fr;

    What it means:
    This creates 3 flexible columns using fr (fractional unit). These columns share the available space in a ratio.
    
Column 1 → 1 part
Column 2 → 2 parts (twice as wide as column 1)
Column 3 → 1 part

    Use Case:
    Use this when you want a **responsive layout** where columns adjust size based on available space.
     

    Example:
    If container width = 800px: → Total parts = 1 + 2 + 1 = 4 → Column widths = 200px, 400px, 200px
    

    ---------------------------------------------------------------------------------
    
grid-template-columns: repeat(4, 180px);

    What it means:
    Instead of writing 180px 180px 180px 180px, we use the repeat() function to make it short.
    
Column 1 → 180px
Column 2 → 180px
Column 3 → 180px
Column 4 → 180px

    Use Case:
    Best when all columns have the **same width** and you want to keep your code cleaner.
    

    ---------------------------------------------------------------------------------
    
grid-template-columns: repeat(3, auto);

    What it means:
     
    This creates 3 columns that automatically adjust based on their content width.
    
Column 1 → auto
Column 2 → auto
Column 3 → auto

    Use Case:
    Best when you want the grid to **shrink or grow to fit the content** inside each column.
    

    ---------------------------------------------------------------------------------

3. grid-template-rows

  • Defines number and height of rows
  • Example: 
    grid-template-rows: 150px 50px 150px 30px 100px;
  • Other formats: 
    
grid-template-rows: repeat(3, 100px);

      
  
    • Creates 3 rows, each of fixed height 100px.
      • 
  
    • Row 1 → 100px
      • 
  
    • Row 2 → 100px
      • 
  
    • Row 3 → 100px
      • 
  
    • Use when you want all rows to be the same height.
      • 

    

    ---------------------------------------------------------------------------------
    
grid-template-rows: 1fr 3fr 1fr;

      
  
    • Creates 3 flexible rows using fr units (fractional space).
      • 
  
    • Row 1 → 1 part
      • 
  
    • Row 2 → 3 parts (largest)
      • 
  
    • Row 3 → 1 part
      • 
  
    • If total height = 500px, result would be 100px, 300px, 100px respectively.
      • 

    

    ---------------------------------------------------------------------------------
    
grid-template-rows: 0.5fr 0.2fr 1fr 0.2fr;

      
  
    • You are creating 4 flexible rows with different proportions:
      • 
  
        
    
      • Row 1 → 0.5 part
        • 
    
      • Row 2 → 0.2 part
        • 
    
      • Row 3 → 1 part (largest row)
        • 
    
      • Row 4 → 0.2 part
        • 
  
      
  
    • Total parts = 0.5 + 0.2 + 1 + 0.2 = 1.9 parts
      • 
  
    • If container height = 950px:
      • 
  
        
    
      • Row 1 → (0.5 / 1.9) × 950 ≈ 250px
        • 
    
      • Row 2 → (0.2 / 1.9) × 950 ≈ 100px
        • 
    
      • Row 3 → (1 / 1.9) × 950 ≈ 500px
        • 
    
      • Row 4 → (0.2 / 1.9) × 950 ≈ 100px
        • 
  
      

    

    Use Case:
    
Use this when you want precise control over row height ratios in a responsive layout.
    


    ---------------------------------------------------------------------------------

4. column-gap, row-gap, gap

  • Controls space between columns and rows
  • gap is shorthand for both
  • Example: 
    
section {
  row-gap: 5px;
  column-gap: 20px;
  /* OR */
  gap: 5px 20px;
}

Grid Item Properties

1. grid-row

  • Shorthand for grid-row-start and grid-row-end
  • Example: 
    grid-row: 1 / 5;
    • This sets the grid item to start on row line 1 and end at row line 5.
    • It will span 4 rows (row lines: 1→2→3→4→5).
    • Used when you want a tall item that covers multiple rows vertically.

2. grid-column

  • Shorthand for grid-column-start and grid-column-end
  • Example: 
    grid-column: 1 / 4;
    • This sets the grid item to start on column line 1 and end at column line 4.
    • It will span 3 columns (column lines: 1→2→3→4).
    • Used when you want a wide item that stretches across multiple columns.

3. grid-area

  • Shorthand for all four:
    • grid-row-start
    • grid-column-start
    • grid-row-end
    • grid-column-end
  • Example: 
    grid-area: 1 / 1 / 3 / 4;
    • This is a shorthand for all four lines: grid-row-start / grid-column-start / grid-row-end / grid-column-end.
    • Starts at row line 1 and ends at row line 3 → spans 2 rows.
    • Starts at column line 1 and ends at column line 4 → spans 3 columns.
    • Same as: 
      
grid-row: 1 / 3;
grid-column: 1 / 4;

Extra Example


.container {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr;
  grid-template-rows: 100px 100px;
  gap: 10px;
}

.item1 {
  grid-column: 1 / 3;
  grid-row: 1 / 2;
}

.item2 {
  grid-area: 2 / 2 / 3 / 4;
}

Note: You can use fr units (fractional space) to create flexible layouts.



Transform

→ Definition:

Transform changes the state of an element like position, rotation, scale, and skew.

→ Syntax:
transform: value;
→ Functions:
1. translate()

Moves an element along the x and y axes.

transform: translateX(200px);
transform: translateY(400px);
transform: translate(200px, 400px);
2. scale()

Increases or decreases the size of an element. increase greater than 1, decrease lesser than 1 same size 1

transform: scaleX(2);     /* Doubles the width */
transform: scaleY(0.5);   /* Halves the height */
transform: scale(2, 0.5); /* Width x2, Height /2 */
3. skew()

Tilts an element along the axes.

transform: skewX(40deg);
transform: skewY(20deg);
transform: skew(40deg, 20deg);
4. rotate()

Rotates an element around its center.

transform: rotate(90deg);   /* Clockwise */
transform: rotate(-90deg);  /* Counterclockwise */
→ Multiple Transforms:
/* Only the last one will apply if written separately */
transform: translate(200px,200px);
transform: scale(2);
transform: rotate(360deg);

/* Correct way */
transform: translate(200px,200px) scale(2) rotate(360deg);

Transition

→ Definition:

Allows CSS property changes to occur smoothly over a duration.

→ Shorthand:
transition: property duration timing-function delay;
→ Properties:
1. transition-property
transition-property: background-color;
2. transition-duration
transition-duration: 2s;
3. transition-delay
transition-delay: 0.5s;
4. transition-timing-function
transition-timing-function: ease;
transition-timing-function: ease-in;
transition-timing-function: ease-out;
transition-timing-function: ease-in-out;
transition-timing-function: linear;
→ Example:
div:hover {
  transition: all 0.5s ease-in-out;
  background-color: red;
}

Box Shadow

→ Definition:

Adds shadow to the box element.

→ Syntax:
box-shadow: x-offset y-offset blur-radius spread-radius color inset;
→ Examples:
box-shadow: 2px 2px 2px 3px blue;
box-shadow: 2px 2px 2px 3px blue inset;

Z-index

→ Definition:

Controls the stack order of elements when they overlap. Only works with positioned elements.

→ Syntax:
z-index: 1;   /* Behind elements with higher z-index */
z-index: 100; /* In front of elements with lower z-index */
→ Note:

Applies only if the element has position set to relative, absolute, fixed, or sticky.



CSS Responsiveness / Media Query

→ Purpose:

To achieve responsiveness, we mainly use the @media rule.

→ What is @media?

The @media rule applies styles based on conditions like screen size, device type, or orientation.

→ Syntax:
@media mediaType mediaOperator mediaFeatures {
  /* CSS Code */
}
→ Key Components:
1. Media Type

Specifies the type of device the CSS applies to.

  • all → Default, applies to all media types. that is for both print and screen.
  • screen → Applies only to screen displays. display on the screen
  • print → Applies to print previews. display when print the page.
2. Media Operator

Combines or negates multiple media features.

  • and → All conditions must be true
  • or → At least one condition must be true
  • not → Negates the condition
3. Media Features

Defines breakpoints based on screen properties.

  • min-width → Minimum width of screen
  • max-width → Maximum width of screen
  • min-height → Minimum height of screen
  • max-height → Maximum height of screen
  • width / height → Compare using <, >, <=, >=
  • orientation → Values: portrait: width>height , landscape: width<height
→ Example: Mobile View
@media screen and (min-width: 0px) and (max-width: 400px) {
  h1::after {
    content: "Mobile View";
  }
  body {
    background-color: teal;
  }
}
→ Example: Tablet View
@media screen and (min-width: 400px) and (max-width: 800px) {
  h1::after {
    content: "Tablet View";
  }
  body {
    background-color: cadetblue;
  }
}


CSS Animation

→ What is Animation?

Animation allows an element to smoothly change from one style to another over time.

It is a shorthand property for:

  • animation-name
  • animation-duration
  • animation-delay
  • animation-timing-function
  • animation-direction
  • animation-iteration-count
→ @keyframes Rule

The @keyframes rule defines the sequence of styles in an animation.

→ Syntax:
@keyframes animationName {
  0% {
    /* initial style */
  }
  50% {
    /* mid-point style */
  }
  100% {
    /* final style */
  }
}
→ animation-name:

Specifies the name of the animation (should match the @keyframes identifier).

→ animation-duration:

Sets how long the animation will run.

  • Default: 0s (animation won't run)
  • Required for animation to work
→ animation-delay:

Specifies the delay before the animation starts.

  • Default: 0s (starts immediately)
→ animation-timing-function:

Defines the speed curve of the animation.

  • linear → constant speed
  • ease → slow start, fast middle, slow end
  • ease-in → slow start
  • ease-out → slow end
  • ease-in-out → slow start and end
→ animation-direction:

Controls the direction of the animation cycle.

  • normal → forward (default)
  • reverse → backward
  • alternate → forward then backward
  • alternate-reverse → backward then forward
→ animation-iteration-count:

Specifies how many times the animation runs.

  • Any number (e.g., 3)
  • infinite → animation runs forever


CSS object-fit

→ Purpose:

The object-fit property is used to specify how an <img> or <video> should be resized to fit its container.

→ Syntax:
object-fit: value;
→ Values:
  • fill → Default. The image is resized to fill the given dimension (stretched or squished if necessary).
  • contain → Keeps the aspect ratio, resized to fit within the container.
  • cover → Keeps the aspect ratio, fills the container, and may be clipped.
  • none → Image is not resized at all.

CSS opacity

→ Purpose:

Sets the transparency level of an element. A value from 0.0 (completely transparent) to 1.0 (fully opaque).

→ Syntax:
opacity: 0 to 1;
→ Note:

Commonly used with the :hover selector to create mouse-over effects.

CSS float

→ Purpose:

The float property is used to position an element to the left or right, allowing text/content to wrap around it.

→ Syntax:
float: left | right | none;
→ Values:
  • left → Floats the element to the left.
  • right → Floats the element to the right.
  • none → Default. No floating.
→ Notes:
  • Floated elements are taken out of normal flow.
  • Text and inline elements wrap around floated elements.

CSS clear

→ Purpose:

Used to prevent elements from flowing next to floated elements. Forces them to move below.

→ Syntax:
clear: left | right | both | none;
→ Values:
  • none → Default. No clearing.
  • left → Clears content floated to the left.
  • right → Clears content floated to the right.
  • both → Clears both left and right floats.
