Web Servers, Hosting and Cloud Fundamentals

A web server is a software program that listens for HTTP/HTTPS requests from clients and sends responses back. It acts as an intermediary between users and applications.

What is a Server?

The term "server" can refer to both a physical machine and a software program.

  • Hardware Server - Physical computer with CPU, RAM, Storage and Network.
  • Software Server - Program that serves requests such as Apache, Nginx or Flask server.

Hardware Server
      ↓

      ↓
Web Server (Apache/Nginx)
      ↓

      ↓
Database

What is Apache and Nginx?

Apache and Nginx are web servers used to manage HTTP and HTTPS traffic between users and applications.

  • Accept incoming requests.
  • Forward requests to applications.
  • Handle HTTPS encryption.
  • Serve static files.
  • Perform load balancing.
  • Manage concurrent connections.
  • Act as reverse proxies.

Simple Definition

Apache/Nginx are traffic managers for web applications.

Why Apache/Nginx Exist?

Applications can serve users directly, but as traffic increases specialized traffic management software becomes useful.

  • Handle many users efficiently.
  • Manage HTTP and HTTPS requests.
  • Provide load balancing.
  • Serve static files faster.
  • Improve reliability.
  • Support reverse proxy architecture.

Hosting a Flask Application


from flask import Flask

app = Flask(__name__)

@app.route("/")
def home():
    return "Hello World"

app.run(host="0.0.0.0", port=5000)

Using host="0.0.0.0" allows devices outside the local machine to access the application.

Apache Installation

Ubuntu/Debian


sudo apt update
sudo apt install apache2

Check Status


sudo systemctl status apache2

Start Apache


sudo systemctl start apache2

Stop Apache


sudo systemctl stop apache2

Restart Apache


sudo systemctl restart apache2

Enable Auto Start During Boot


sudo systemctl enable apache2

Apache Important Directories

Path Purpose

Apache Virtual Host Configuration

Virtual Hosts allow multiple websites to run on the same server. 


sudo nano /etc/apache2/sites-available/myapp.conf
    

<VirtualHost *:80>
    ServerName example.com
    DocumentRoot /var/www/myapp
</VirtualHost>

Enable Site


sudo a2ensite myapp.conf
sudo systemctl reload apache2

Nginx Reverse Proxy Configuration

Most modern applications run behind Nginx using reverse proxy. 


sudo nano /etc/nginx/sites-available/myapp
    

server {
    listen 80;

    server_name example.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
    }
}

Enable Site


sudo ln -s /etc/nginx/sites-available/myapp \
/etc/nginx/sites-enabled/

Test Configuration


sudo nginx -t

Reload Nginx


sudo systemctl reload nginx

HTTPS Configuration

HTTPS encrypts communication between users and servers.

Install SSL Tool


sudo apt install certbot

Nginx SSL


sudo certbot --nginx

Apache SSL


sudo certbot --apache

Certbot automatically generates SSL certificates and configures HTTPS.

Load Balancing in Nginx


upstream backend {

    server 127.0.0.1:5000;
    server 127.0.0.1:5001;
    server 127.0.0.1:5002;

}

server {

    location / {
        proxy_pass http://backend;
    }

}

Nginx distributes requests across multiple backend servers.
