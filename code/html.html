Heading Tags

Definition: Heading tags (<h1> to <h6>) are used to define titles or subtitles in a webpage. They represent six levels of headings where <h1> is the largest and most important, and <h6> is the smallest and least important.

Syntax: 

          <h1> Heading 1 </h1>
          <h2> Heading 2 </h2>
          <h3> Heading 3 </h3>
          <h4> Heading 4 </h4>
          <h5> Heading 5 </h5>
          <h6> Heading 6 </h6>

Notes:

  • They are block-level elements (occupy full width).
  • Heading tags are important for SEO (Search Engine Optimization), as <h1> usually represents the main title of a webpage.
  • You should use them in a hierarchical order (h1 → h2 → h3 ...).
  • Default sizes may vary across browsers, but approximately:
    • <h1> → 2em (32px)
    • <h2> → 1.5em (24px)
    • <h3> → 1.17em (18.72px)
    • <h4> → 1em (16px)
    • <h5> → 0.83em (13.28px)
    • <h6> → 0.67em (10.72px)

Element

Definition: An HTML element is a combination of an opening tag, content, and a closing tag.

Syntax:

<tagname> content </tagname>

Types of Elements:

1. Inline Elements

Definition: Inline elements do not start on a new line. They only take up as much width as necessary. You cannot set height/width easily. Displayed in Sameline and we cannot adjust Height and Width.

Eg: <b>, <i>, <mark>, <span>, <a>, <sub>, <sup> etc.

2. Block-level Elements

Definition: Block elements always start on a new line and take up the full width available. Their height and width can be adjusted. It will take the entire width of the page and can able to adjust the Height and Width.

Eg: <h1>, <h2>, <p>, <div>, <section> etc.

3. Inline-Block Elements

Definition: Inline-block behaves like inline (stays in same line) but allows setting width and height like block elements. Combination of both display in same line and we can adjust height and width.

Eg: <img>, <input>, <button>, <select> etc.

HTML Forms - Detailed Explanation

1. Basic Form Structure

The <form> element is used to collect user input and send it to a server or handle it with JavaScript. Important attributes:

  • action – URL where the form data is sent.
  • method – GET (default) or POST, defines how data is sent.

GET vs POST in HTML Forms

1. Definition

In HTML, forms are used to send data from the client (browser) to the server. The method attribute of the <form> tag specifies how the data should be sent. The two most common methods are:

  • GET: Appends the form data into the URL as query parameters.
  • POST: Sends the form data in the request body, not visible in the URL.

2. Syntax

<form action="server_page.php" method="get">
    <input type="text" name="username">
    <input type="submit">
</form>

<form action="server_page.php" method="post">
    <input type="text" name="username">
    <input type="submit">
</form>

3. Key Differences

Aspect GET POST
Purpose Primarily used for retrieving data from the server. Primarily used for submitting data to the server to create or modify resources.
Data Visibility Data is visible in the URL (e.g., ?username=John) Data is hidden, sent in the request body
Use Case Fetching or requesting data (e.g., search engines, filters) Submitting sensitive data (e.g., login, signup forms)
Data Length Limited by URL length (about 2000 characters) No restriction on data length
Security Less secure (data can be bookmarked, cached, logged) More secure (data not shown in URL)
Performance Faster, as data is appended to URL Slightly slower due to larger request
Idempotency Safe to repeat (does not change server state) Used for actions that modify server state

The <textarea> element in HTML is used to create a multi-line text input field. It is commonly used in forms where users can enter longer pieces of text such as comments, feedback, or descriptions.

📌 Syntax

<textarea name="fieldName" rows="number" cols="number">Default Text</textarea>

📌 Important Attributes

  • name → Specifies the name of the textarea (used when submitting form data).
  • rows → Defines the visible number of text lines.
  • cols → Defines the visible width of the textarea (number of characters per line).
  • placeholder → Provides a hint text inside the textarea until the user types something.
  • maxlength → Restricts the maximum number of characters allowed.
  • readonly → Makes the textarea non-editable but still selectable.
  • disabled → Disables the textarea so the user cannot interact with it.
  • required → Makes filling the textarea mandatory before form submission.
  • autofocus → Automatically puts the cursor inside the textarea when the page loads.
  • wrap → Controls text wrapping:
    • soft → Text wraps visually but not stored with line breaks.
    • hard → Line breaks are inserted and submitted as part of the value.

HTML Non-Semantic Element

In HTML, elements can be categorized as semantic or non-semantic:

  • Semantic Elements: These elements have a specific meaning and clearly describe their purpose both to the browser and the developer.
    Examples: <header>, <footer>, <article>, <section>, <nav>.
  • Non-Semantic Elements: These elements don’t convey any meaning about the content they contain. They are used mainly for styling or grouping content together.
    Examples: <div>, <span>.

About <div> Element

The <div> (division) element is a block-level container used to group other HTML elements. It has no semantic meaning by itself but is very useful for applying CSS styles or JavaScript functionality.

Syntax:

<div>
    Content here...
</div>

Key Points:

  • <div> is a block-level element (starts on a new line).
  • Used to group multiple elements for styling and layout purposes.
  • Often used as a container for sections of a page.
  • Does not provide any meaning about the content inside.

Example:

<div style="border: 1px solid black; padding: 10px;">
    <h2>This is inside a div</h2>
    <p>The div groups this paragraph and heading together.</p>
</div>

HTML Semantic Elements

Semantic elements in HTML are those that clearly describe their meaning in a human- and machine-readable way. Unlike <div> or <span> (which are non-semantic), semantic elements clearly define their purpose and structure of the web page. They make the code more accessible, especially for search engines and screen readers.

Common Semantic Elements

  • <header> → Defines the header section of a page or article (usually contains logo, navigation, or titles).
  • <nav> → Represents a navigation section that links to other parts of the site.
  • <main> → Specifies the main content of the document (unique per page).
  • <section> → Defines a thematic grouping of content (like chapters or parts of a webpage).
  • <article> → Represents independent, self-contained content (like a blog post, news article).
  • <aside> → Defines content indirectly related to the main content (like a sidebar, ads, or related links).
  • <footer> → Defines the footer of a page or section (usually contains copyright, contact info, links).

Syntax Example

<header>
     <h1>This is Header</h1>
     <nav>
          <a href="#">Home</a>
          <a href="#">Explore</a>
          <a href="#">Contact</a>
     </nav>
</header>

<main>
     <section>
          <h1>This is Section 1</h1>
          <article><h1>This is Article 1</h1></article>
          <article><h1>This is Article 2</h1></article>
     </section>

     <aside>
          <h1>This is Aside</h1>
     </aside>
</main>

<footer>
     <h1>This is Footer</h1>
</footer>

Output Explanation

  • The Header appears at the top with navigation links.
  • The Main section contains multiple Sections and Articles.
  • The Aside is displayed as a sidebar (often styled separately with CSS).
  • The Footer appears at the bottom with additional info.
