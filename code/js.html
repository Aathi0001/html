Type Conversion (Type Casting) in JavaScript

Type conversion means changing a value from one data type to another.

In JavaScript, this happens in two ways:

  • Implicit Conversion (Type coercion) ‚Üí done automatically by JavaScript at runtime.
  • Explicit Conversion ‚Üí done manually by the developer. It might lead to data loss.

1. Implicit Conversion (Type Coercion)

JavaScript automatically converts values when needed.

// String + Number ‚Üí String
console.log('5' + 2); // "52"

// Number + Boolean ‚Üí Number
console.log(5 + true); // 6 (true ‚Üí 1)

// String * Number ‚Üí Number
console.log('4' * 2); // 8

// Chained comparisons
console.log(1 < 2 < 3); // true
// Explanation: (1 < 2) ‚Üí true ‚Üí true is converted to 1 ‚Üí (1 < 3) ‚Üí true

console.log(3 > 2 > 1); // false
// Explanation: (3 > 2) ‚Üí true ‚Üí true is converted to 1 ‚Üí (1 > 1) ‚Üí false

// Boolean addition
console.log(true + true); // 2 (true ‚Üí 1, so 1 + 1 = 2)

// String minus number
console.log("abc" - 5); // NaN (string cannot be converted to a valid number)

2. Explicit Conversion

We manually convert using built-in functions.

To Number:

Number("42");   // 42
Number("3.14"); // 3.14
Number(true);   // 1
Number(false);  // 0

To String:

String(42);         // "42"
String(true);       // "true"
String(null);       // "null"
(42).toString();    // "42"

To Boolean:

Boolean(1);    // true
Boolean(0);    // false
Boolean("");   // false
Boolean("hi"); // true

Special Notes:

  • parseInt() ‚Üí Converts string to integer.
  • parseFloat() ‚Üí Converts string to decimal number.
  • Number() ‚Üí Converts its arguments into number.
  • String() ‚Üí Converts its arguments into number.
  • Boolean() ‚Üí Converts its arguments into boolean.
  • +value ‚Üí Quick way to convert to number.
  • == ‚Üí Allow conversion.
  • === ‚Üí Does not allow conversion.

Example:

console.log(Number("123"));     // 123
console.log(String(456));       // "456"
console.log(Boolean("Hello"));  // true
console.log(parseInt("99px"));  // 99

    üéØ Event Capturing vs Bubbling in JavaScript

    When we add an event listener in JavaScript, it can run in either the bubbling phase (default) or the capturing phase.

    üìù Syntax

    element.addEventListener(event, handler, useCapture);
    • event ‚Üí the event type (e.g., "click").
    • handler ‚Üí the function to run.
    • useCapture ‚Üí false (default ‚Üí bubbling) OR true (capturing).

    üìå Explanation

    • false (default ‚Üí Bubbling Phase): Event goes down to the target, runs on the target, then bubbles up to ancestors.
    • true (Capturing Phase): Event runs on ancestors first, while traveling down, before reaching the target.

  console.log("Outer - Bubbling (runs LAST)");

</script>

    • To intercept/block events before children handle them.
    • To create global rules (e.g., disabling all child clicks).

    üìå JavaScript Events ‚Äî Event Flow

    Event Flow in JavaScript describes the order in which events are received on the DOM elements. There are two main phases:

    • Capturing Phase (Event travels from window ‚ûù document ‚ûù html ‚ûù body ‚ûù parent ‚ûù child)

    üåü JavaScript Promise

    üìå Definition

    A Promise in JavaScript is an object that represents the eventual completion (success) or failure of an asynchronous operation.

    üìù DOM Forms & Input Handling

    Forms allow user input. JavaScript can capture, validate, and process this data. We use value, checked, selected, and events like input, change, and submit.

    üìå Form Events

    • input ‚Üí Fires on every keystroke/change.
    • change ‚Üí Fires when element loses focus & value is changed.
    • focus ‚Üí Fires when input gets focus.
    • blur ‚Üí Fires when input loses focus.
    • submit ‚Üí Fires when form is submitted.
